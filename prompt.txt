BOT MAQSADI
Bot maqsadi Instagram reels orqali kino tarqatish. Flow quyidagicha: Instagram da kino traileri reeli joylanganda, caption da #KOD bo'ladi (masalan #MOV4829). Meta Graph Webhook script caption dan kodni va reel linkini ajratib, bot API ga yuboradi. Bot shu reel_link + movie_code ni reels jadvaliga saqlaydi. Foydalanuvchi reel linkini botga yuborsa, bot reels jadvalda qidiradi, movie_id topadi va videoni yuboradi. Video Telegram private kanalda file_id sifatida saqlanadi — bot sendVideo(file_id) bilan yuboradi. Kanal linki hech qachon userga ko'rinmaydi, fayl yuklanmaydi.

NIMA QILISH KERAK — TO'LIQ REFACTOR

1. /start COMMAND — UX REFACTOR
Hozirgi muammo: /start oddiy xabar yuboradi, g'alati hardcoded tekst bor.
Kerakli flow: /start kelganda agar admin bo'lsa "🎬 Movie Linker — Admin Panel\nWelcome back, {username}!" xabari chiqadi, agar oddiy user bo'lsa "🎬 Movie Linker\nInstagram reels orqali kinolarni toping!\n\nTilni tanlang:" xabari chiqadi. Har ikki holatda ham til tanlash inline keyboard chiqadi: [ 🇺🇿 UZ ] [ 🇷🇺 RU ] [ 🇬🇧 EN ]. Callback data lar: lang_uz, lang_ru, lang_en (hozirgi adm_uz, adm_ru, adm_en emas).

2. TIL TANLASH CALLBACK — ADMIN VA USER UCHUN ALOHIDA
Oddiy user til tanlaganda: til bazaga va Redis cache ga saqlanadi, "✅ Til o'rnatildi. Instagram reel linkini yuboring." xabari chiqadi, oldingi xabar DeleteMessage bilan o'chiriladi. Admin til tanlaganda: til saqlanadi, statistika chiqadi (foydalanuvchilar soni, kinolar soni, kanallar soni), pastda doim turadigan ReplyKeyboard chiqadi: [ 📊 Statistika ] [ 🎬 Kinolar ] / [ 📢 Kanallar ] [ ➕ Kino qo'shish ] / [ 🌐 Web Panel ]. Shuningdek admin uchun SetMenuButtonForChat(chatID, cfg.FrontendURL) chaqiriladi — faqat o'sha admin ning chat_id bilan, global emas.

3. ADMIN REPLYKEYBOARD COMMANDS
📊 Statistika bosilganda: "📊 Bot Statistikasi:\n👥 Jami foydalanuvchilar: {N}\n🎬 Jami kinolar: {N}\n📢 Aktiv kanallar: {N}" chiqadi.
🎬 Kinolar bosilganda: oxirgi 10 ta kino ro'yxati chiqadi formatda "1. MOV4829 — Caption text".
📢 Kanallar bosilganda: aktiv kanallar ro'yxati chiqadi.
➕ Kino qo'shish bosilganda: "📹 Kino qo'shish uchun:\n1. Private kanalga videoni yuboring\n2. Video ustiga bosing → Forward → Menga yuboring\nYoki: /add_movie <file_id> <caption>" xabari chiqadi. Admin video forward qilganda yoki /add_movie yuborilganda bot kinoni bazaga saqlaydi, unikum kod generatsiya qiladi (MOV + 4 xonali random raqam, unique bo'lishi kerak), javob beradi: "✅ Kino qo'shildi!\n🔑 Kod: MOV4829\n📝 Caption: ...".
🌐 Web Panel bosilganda: faqat adminga WebApp tugmasi yuboradi, TMA URL: cfg.FrontendURL.

4. FORCE SUBSCRIBE — INLINE BUTTONS BILAN
Hozirgi muammo: oddiy tekst bilan kanal linkini yozib yuboradi, bu yaxshi emas.
Kerakli ko'rinish: "⚠️ Botdan foydalanish uchun quyidagi kanallarga obuna bo'ling:" xabari chiqadi. Har bir kanal uchun alohida URL inline button chiqadi: [ 📢 https://t.me/channel1 ] (url type button, tashqi link ochadi). Oxirida [ ✅ Tekshirish ] callback button chiqadi (callback_data: "check_sub"). check_sub callback kelganda: Redis subscription cache invalidate qilinadi, qayta checkChannelsMembership chaqiriladi, obuna bo'lgan bo'lsa "✅ Rahmat! Endi reel linkini yuboring." deydi, obuna bo'lmagan bo'lsa "❌ Siz hali barcha kanallarga obuna bo'lmadingiz!" deydi.

client.go da SendInlineKeyboard metodini update qilish kerak: agar btn.URL != "" bo'lsa b["url"] = btn.URL ishlatilsin, agar btn.Data != "" bo'lsa b["callback_data"] = btn.Data ishlatilsin.

6. DATABASE — YANGI JADVALLAR VA O'ZGARISHLAR
movies jadvaliga code field qo'shish: ALTER TABLE movies ADD COLUMN IF NOT EXISTS code VARCHAR(20) UNIQUE; Kod formati: MOV + 4 xonali random raqam, unique bo'lishi shart, generatsiya paytida collision tekshirilsin.
Yangi reels jadvali:
CREATE TABLE IF NOT EXISTS reels (
    id SERIAL PRIMARY KEY,
    reel_link TEXT UNIQUE NOT NULL,
    movie_id INT REFERENCES movies(id) ON DELETE SET NULL,
    linked_at TIMESTAMP DEFAULT NOW()
);
postgres.go ga yangi metodlar: GetMovieByCode(ctx, code) — code bo'yicha movie topadi. SaveReel(ctx, reelLink, movieID) — reels jadvaliga saqlaydi. GetMovieByReelLink(ctx, reelLink) — reel link bo'yicha movie topadi. GenerateUniqueMovieCode(ctx) — MOV + 4 raqam, unique bo'lguncha retry qiladi.

7. USER MESSAGE HANDLING — REEL LINK DETECTION
handleMessage da instagram.com/ bo'lgan xabarlar handleUserMovieRequest ga yuboriladi. handleUserMovieRequest da qidiruv tartibi quyidagicha: avval reels jadvalda reel_link bo'yicha qidiradi, movie_id topsa video yuboradi; topilmasa movies jadvalda instagram_url bo'yicha qidiradi (legacy support); topilmasa "❌ Kino topilmadi." xabari yuboradi.

8. META WEBHOOK ENDPOINT — YANGI HTTP ENDPOINT
handler.go da yangi endpoint: POST /api/meta/reel
Request body:
{
  "reel_link": "https://www.instagram.com/reel/ABC123/",
  "movie_code": "MOV4829",
  "auth_token": "secret_token"
}
Auth tekshiruvi: Authorization: Bearer {META_WEBHOOK_SECRET} header yoki body da auth_token. Logic: auth_token tekshiriladi (cfg.MetaWebhookSecret bilan), movie_code bo'yicha movies jadvaldan movie_id topiladi, reels jadvaliga (reel_link, movie_id) saqlanadi, 200 OK qaytariladi. main.go da route qo'shiladi: mux.HandleFunc("/api/meta/reel", botHandler.MetaReelHandler).

9. SETMENUBUTTON — FAQAT ADMIN UCHUN, TO'G'RI URL BILAN
Hozirgi muammo: bot start bo'lganda global SetMenuButton chaqiriladi va cfg.WebhookURL ishlatiladi — bu noto'g'ri, chunki WebhookURL backend URL, lekin TMA uchun frontend URL kerak, va menu button barcha userlarga ko'rinib qoladi.
Kerakli: NewBotService ichidagi global SetMenuButton chaqiruvini o'chirish kerak. Uning o'rniga SetMenuButtonForChat(chatID int64, webAppURL string) metodi qo'shiladi. Bu metod faqat admin til tanlaganda chaqiriladi, o'sha admin ning chat_id bilan. client.go da metod:
func (c *TelegramClient) SetMenuButtonForChat(chatID int64, webAppURL string) error {
    apiURL := fmt.Sprintf("%s/setChatMenuButton", c.apiURL)
    payload := map[string]interface{}{
        "chat_id": chatID,
        "menu_button": map[string]interface{}{
            "type": "web_app",
            "text": "🎬 Admin Panel",
            "web_app": map[string]interface{}{
                "url": webAppURL,
            },
        },
    }
    return c.postJSON(context.Background(), apiURL, payload)
}
